# Release Notes: 2026-06-09

## `2026.6.9`

C/C++ integer-width standard.

- Adds a C/C++ rule requiring fixed-width integer spellings in owned code.
- Requires C++ profiles to configure clang-tidy `google-runtime-int`.
- Requires that check to be listed under `WarningsAsErrors` for release-facing
  enforcement.
- Documents the normal owned-code spellings: `std::int32_t`, `std::uint32_t`,
  `std::int64_t`, and `std::uint64_t`.
- Keeps external-library integer spellings allowed at external API boundaries
  when converted into owned types at the boundary.
