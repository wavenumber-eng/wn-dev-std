# 2026-07-15

Release version: `2026.7.15`

## Changes

- Improves human-facing CLI text output for plan, log, ADR, and requirement
  reads with padded terminal spacing, state summaries, readable grouping,
  wrapped step and record descriptions, and TTY color treatments that keep
  plan ids, step ids, statuses, and section headings easier to scan.
- Keeps JSON output as the stable machine-readable interface while improving
  text output for human review.
- Adds strict Rack test-suite governance through the `tests` audit scope. The
  audit checks configured test roots, `rack.toml` strata, `STRATUM.toml`
  subtest manifests, signoff strata, and discovered `test_*.py` files against
  committed reality.
- Adds the `docs.test_strategy` audit scope and canonical
  `docs/test-strategy.html` document so packages describe high-level test
  architecture, parity lanes, fixture and oracle strategy, coverage evidence,
  missing/orphaned assets, and signoff boundaries beyond Rack manifests.
- Adds required plan closeout review for new test runtime impact through the
  `test-runtime-impact-audit` step and exit criterion before external review.
  Generated plans use recommended titles, while audits treat the ids and
  external-review dependency order as canonical.
- Documents the migration impact for existing active plans: migrate active
  plans to include the new runtime-impact hook, pin CI to the previous reviewed
  release, or defer `docs.plans` adoption until the catalog is updated.
- Adds CI guidance for GitHub Actions and GitLab CI that runs dev-std
  governance first and gates expensive matrix jobs behind that result.
- Wires this repository's CI and release workflows to run
  `dev-std audit .` before Rack signoff, packaging, and publishing.
- Expands the config schema so it matches the current implementation surface:
  `standard_version`, package/workspace kind, `enabled_scopes`, workspace
  members, Rack test governance, generated governance output, plan ignores,
  compatibility pruning, and PR hygiene.
- Removes company-specific wording from public CLI-adjacent docs/templates so
  the tool reads as generic development-standards governance tooling.
- Retires the temporary issue-19 plan chain after independent review and full
  Rack signoff.

## Validation

- Full Rack signoff passed locally: 23/23 subtests, 274 tests passed, 0 failed,
  1 skipped.
- `dev-std audit .` and `dev-std audit . --scope docs.plans` passed after plan
  closeout, with no active plans or logs remaining.
