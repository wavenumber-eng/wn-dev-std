# Release Notes: 2026-06-27

## `2026.6.27`

Plan and work-log hygiene update.

- Adds `dev-std log show <log_id>` with text and JSON output so agents can read
  an attached work-log body directly after listing logs for a plan.
- Tightens `docs.plans` audits so Markdown files under approved plan roots must
  be compliant plan/log documents or `README.md`.
- Adds structured plan step metadata and validation for active, pending,
  blocked, and done step states.
- Adds structured plan exit-criteria metadata so `plan list`, `plan show`, and
  `docs.plans` can answer whether a plan is ready for closeout.
- Documents the package-local audit boundary, root discovery rules, and
  non-destructive plan/log command surface.
