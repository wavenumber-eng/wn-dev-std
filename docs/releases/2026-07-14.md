# 2026-07-14

Release version: `2026.7.14`

## Changes

- Adds `standard_version` config alignment checks, configured audit scopes,
  workspace member aggregation, and optional PyPI upstream-version warnings.
- Runs only the checks needed for requested audit scopes so large workspace
  audits do not pay setup cost for unrelated governance areas.
- Keeps upstream-version probes warning-only when PyPI or an intermediary
  returns malformed HTTP protocol responses.
- Moves browser/web policy checks into a dedicated module without changing
  their behavior.
- Documents pinned PyPI `wn-dev-std` usage for downstream governance CI and
  repairs the local release policy guidance.
- Adds the accepted JSON Contract Standard for durable JSON object identity:
  root `type` plus `version`, nested `kind`, documented schema-labeled
  compatibility files, JSON Schema artifacts, and Pydantic/FastAPI usage.
- Requires active/pending/blocked plans to include a closeout governance-doc
  audit and external review as both structured steps and exit criteria, with
  final review ordered after the design-doc/ADR/requirement alignment
  checkpoint.
- Fails attempted plan closeout states such as `done`, `closed`, `complete`,
  and `finished` with guidance to move durable information into design docs,
  ADRs, requirements, tests, or release notes before deleting the active plan.
- Updates `dev-std plan create` to emit compliant plan defaults.
- Documents vendor-manifest compatibility, signoff checklist expectations,
  Python 3.12 pin rationale, and squash/no-linear-history public PR policy.
- Adds tests that keep the new standards guidance, scoped-dispatch behavior,
  and plan-policy enforcement from regressing.
