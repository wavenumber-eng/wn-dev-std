## Policy Boundary

The adapter uses Rack non-strict audit mode in this slice. That preserves the
existing dev-std manifest-drift contract while avoiding an immediate migration
requirement for Rack strict metadata fields such as `test_cases` and
`test_case_type`. Strict metadata remains available for a later policy decision
after runtime and migration impact are reviewed.
